# Status vs State

## Schematics

*No suitable open-license schematics found on Wikimedia Commons for this topic.*

## Definition

**Status** refers to a categorical label or classification assigned to something (e.g., active, complete, pending), answering *"what category is it in?"*. **State** describes the actual condition, configuration, or snapshot of something at a given moment, answering *"what does it look like right now?"*. Confusing the two is a common B2-level error because many languages use a single word for both.

## Key Points

- **Status = label:** a discrete tag from a predefined set — think dropdown menus: `active | on-hold | done`
- **State = snapshot:** the full picture of how something currently exists — contents, arrangement, configuration
- A folder has a **status** of "reviewed" but its **state** is "containing three files and two subfolders"
- [[English Collocations]] to remember: *final state* (not ~~final status~~) when describing what something looks like; *project status* (not ~~project state~~) when reporting progress category
- In technical writing, *state* often pairs with verbs like *capture*, *preserve*, *restore*; *status* pairs with *check*, *update*, *report*

## Examples

- "The **status** of the deployment is *failed*." (categorical label)
- "The **state** of the temporary folder should comprise one implementation file and one review file." (describing actual contents)
- "What's the **status** of your visa application?" → expecting a label: submitted, under review, approved
- "Describe the **state** of the experiment." → expecting a snapshot: detector configuration, beam energy, target position
