# .gitignore

## Definition

A `.gitignore` file tells Git which files and directories to **exclude from tracking**. It prevents matching paths from appearing in `git status`, being staged, or committed — but it does not preserve, back up, or restore those files in any way.

## Key Points

- `.gitignore` is a **filter, not a vault** — ignored files exist only on your local disk and are never saved to the repository
- If an ignored file is deleted (e.g., by a [[Git Submodules|submodule]] re-initialization or `git clean`), it is gone permanently
- Patterns support globs: `.venv/`, `*.pyc`, `__pycache__/`, `node_modules/`
- A file already tracked by Git is **not affected** by adding it to `.gitignore` — you must `git rm --cached` it first
- Each directory can have its own `.gitignore`; rules cascade from parent to child

## Examples

- `.venv/` in Marker's `.gitignore` prevents the virtual environment from being committed, but also means it disappears after a fresh clone
- `*.pyc` prevents compiled Python bytecode from cluttering the repo
