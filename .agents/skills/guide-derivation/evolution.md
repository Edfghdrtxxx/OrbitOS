# Guide-Derivation — Evolution Log

## 2026-03-10
### Lessons
- When the user asks questions relevant to the derivation (e.g. motivation, meaning of terms, why a step is needed) during a **learn-mode or quick-mode** session, record the Q&A pairs in the derivation note as a callout (e.g. `> [!faq]`) so they persist for future review — don't leave them only in terminal output.
- **Dual-explanation mode (Feynman):** When the user requests "Feynman" or "explain like I'm a child," provide **two layers** for each step and FAQ: (1) the academic/rigorous explanation, and (2) a Feynman-style analogy in plain language. Both go into the note — the academic version ensures exam-readiness, the Feynman version ensures genuine understanding. This can be combined with either quick or learn mode.
- **AI-driven explanations on request:** If the user explicitly asks the AI to answer/explain rather than be quizzed (Socratic), switch from learn-mode questioning to AI-driven presentation. The user reads in Obsidian. This is not the same as quick mode — it's slower, step-by-step, with physical insight and analogies at each step, but the AI drives instead of asking.
