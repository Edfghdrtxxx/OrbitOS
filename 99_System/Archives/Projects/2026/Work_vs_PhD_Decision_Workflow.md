## 2) Hard Gates (Pass/Fail Before Scoring)

If a track fails any hard gate, it cannot be selected as the primary track.

1. **Legal Feasibility Gate**
   - Current-cycle official requirements are met and documented.
2. **Timeline Gate**
   - Path can be realistically executed around graduation in **June 2027**.
3. **Financial Safety Gate**
   - Work track: enough runway for job search and relocation.
   - PhD track: confirmed scholarship/funding or sustainable self-funding plan.
4. **Lifestyle Gate (Anti-996)**
   - Expected day-to-day working/studying pattern is compatible with your non-996 boundary.

Track hard-gate status is maintained in `[[Work_vs_PhD_Scorecard]]`.

## 3) Weighted Score Model (Comprehensive Comparison)

### Rating and Confidence

- Rating scale per criterion: `1` (very weak) to `5` (very strong)
- Confidence scale: `0.50` to `1.00`
  - `1.00`: officially verified and current
  - `0.75`: mostly verified, minor gaps
  - `0.50`: significant uncertainty

### Calculation

- Raw score: `sum(weight * rating / 5)`
- Confidence-adjusted score: `sum(weight * rating / 5 * confidence)`
- Use confidence-adjusted score as the final ranking basis.

## 4) Workflow Cadence

1. **Define comparison scope**
   - Lock both options as `Work Track` vs `PhD Track`.
   - Freeze evaluation criteria and weights (no mid-cycle edits unless explicitly logged).
2. **Collect evidence**
   - Update Work/PhD checklist notes with source-backed facts.
   - Add missing official references into `[[Official_Source_Verification_2026-02-13]]` (or a newer dated verification note).
3. **Run scoring round**
   - Fill `[[Work_vs_PhD_Scorecard]]` with rating + confidence + evidence link for each criterion.
4. **Run stress test**
   - Evaluate 3 scenarios: `Best case`, `Base case`, `Worst case`.
   - Check whether the winner remains stable across scenarios.
5. **Decision checkpoint**
   - Use the decision rule in section 5.
   - Record final choice and rationale in section 6.
6. **Commit action plan**
   - Convert the chosen primary track into concrete next 90-day tasks.
   - Keep backup-track minimum-maintenance tasks active.

## 5) Decision Rule (How to Actually Choose)

1. Compare confidence-adjusted totals.
2. Apply this exact rule:
   - If score gap is `>= 8` points and both tracks have average confidence `>= 0.75`, finalize the higher score as primary.
   - Otherwise, set decision status to `Revise` and run another evidence cycle.
3. Tie-break order (only when still close after re-score):
   1. Higher immigration certainty by `2030-12-31`
   2. Better anti-996 compatibility
   3. Stronger long-term ML/AI career positioning in Japan
   4. Better downside protection if first attempt fails
